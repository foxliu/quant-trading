# 一、Risk Context / Risk Engine 的工程定位（冻结）

## 1️⃣ 在整体流水线中的位置

```text
Strategy
  └── Signal (TargetQty)
        ↓
Planner
  └── Order (未修正)
        ↓
Position Engine
  └── Order (Delta Qty + Side)
        ↓
Risk Engine   ←【当前阶段】
  ├── 校验是否允许
  ├── 裁剪（Clamp）
  └── 决定是否放行
        ↓
Execution（后续）
```

---

## 2️⃣ Risk Engine 的职责（V1 冻结）

**只做三类事情：**

1. **限制类风控（Limit）**

    * 最大仓位
    * 单笔最大下单量
2. **合法性校验（Validation）**

    * 数量是否为正
    * Side 是否存在
3. **放行 / 拒绝**

    * 不修改仓位事实
    * 不做交易所逻辑

**明确不做：**

* 不计算 Target → Delta（已在 Position）
* 不更新 Position
* 不关心成交
* 不访问 Account Balance（留给下一阶段）

# 二、Risk Context（domain / application）

## 风控配置本质

Risk Context 不是“当前状态”，而是：

> **一组可变但稳定结构的约束条件**

---

## `internal/application/risk/context.go`

```go
package risk

/*
Context
=======

Risk Context 表示一组风控规则的集合。
*/
type Context struct {
	MaxPositionQty float64 // 单标的最大绝对仓位
	MaxOrderQty    float64 // 单笔最大下单量
}
```

```go
func NewContext() *Context {
	return &Context{
		MaxPositionQty: 0, // 0 = 不限制
		MaxOrderQty:    0, // 0 = 不限制
	}
}
```

# 三、Risk Engine 接口（冻结）

我们沿用你之前已经使用并确认过的接口形式，仅扩展语义。

## `internal/application/risk/engine.go`

```go
package risk

import (
	"context"

	"quant-trading/internal/domain/order"
)
```

```go
/*
Engine
======

Risk Engine 的职责（V1）：

- 接收 Position Engine 输出的 Order
- 进行风控校验与裁剪
- 放行或拒绝
*/
type Engine interface {
	Start(ctx context.Context) error
	Stop() error

	Consume(order.Order)
}
```

> 说明
>
> * `Consume` 是 **非阻塞入口**
> * 是否通过由内部决定
