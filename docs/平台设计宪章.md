# 量化交易平台设计宪章

## 一、宪章目的（Purpose）

本宪章用于**长期约束与统一**量化交易平台的整体设计方向与工程决策。

它不是阶段性方案，也不是实现说明，而是一组**高于具体架构与代码的设计共识**。

**除非平台战略发生根本性变化，否则本宪章不应被轻易修改。**

---

## 二、平台北极星定义（North Star）

> **我们要打造的是：**
> 一个**支持多策略并行运行、统一管理多账户，并可覆盖股票 / 期货 / 期权的通用量化交易平台**。

该平台应具备以下长期能力：

* 多策略同时运行，互相隔离
* 多账户统一调度与风险约束
* 跨资产类型（股票 / 期货 / 期权）的一致策略体验
* 同一策略代码可运行于回测与实盘环境

**所有设计与工程实现，必须服从这一北极星目标。**

---

## 三、设计公理（Design Axioms）

以下公理是平台级**不可变约束**。

### 公理一：策略、账户、资产、交易所必须强解耦

* Strategy 表达的是**交易意图与决策逻辑**
* Account 是可调度、可仲裁的资源
* Asset / Instrument 描述金融合约属性
* Exchange / Broker 属于基础设施实现

任何层级**不得同时承担多个角色的职责**。

---

### 公理二：策略永远运行在受控上下文中

* Strategy **不能直接访问**账户、交易所或基础设施
* Strategy 的唯一交互入口是 `Strategy Context`
* Context 提供的是**能力接口**，而不是资源实体

> Context = 策略的“世界观边界”

---

### 公理三：多策略是系统的一等公民

* 每个 Strategy 都必须拥有独立的 Runtime
* Runtime 负责策略的生命周期与隔离
* Engine 的职责是**管理 Runtime 集合**，而非直接驱动策略逻辑

禁止使用共享全局状态支撑多策略运行。

---

### 公理四：资产类型差异必须被抽象吞噬

* 股票 / 期货 / 期权的差异
* 合约到期、保证金、行权逻辑

这些差异**只能存在于资产与交易抽象层**，不得泄漏至 Strategy 或 Engine。

> 策略必须对资产类型保持“无感”。

---

### 公理五：账户是资源，不是配置

* 账户具有状态、风险约束与并发访问问题
* 账户必须被统一调度与仲裁

任何将账户简化为静态配置的设计，均视为方向性错误。

---

### 公理六：回测与实盘的差异只能存在于上下文实现层

* Strategy 接口必须完全一致
* Strategy 代码在回测与实盘环境中不允许分叉

唯一允许差异的地方：

* Context 的具体实现
* 行情源 / 撮合与执行适配器

---

## 四、主上下文的宪章级裁定

### 主上下文定义

> **主上下文（Main Context） = 策略运行与调度内核**

### 主上下文允许的职责

* 策略生命周期管理
* 策略运行调度（Bar / Event 驱动）
* Context 构建与注入
* 策略隔离与并行运行

### 主上下文永久禁止的职责

* 账户与资金管理
* 资产类型判断逻辑
* 交易所 / Broker 适配
* 回测 / 实盘条件分支判断

主上下文一经冻结，不再因短期实现便利而扩展职责。

---

## 五、演进原则（Evolution Rules）

* 不通过修改主上下文来承载新能力
* 所有新能力必须以**新增上下文或资源模型**的方式引入
* 禁止“先写死、后重构”的设计路径

任何违反本宪章的设计，应被视为**平台级设计缺陷**。

---

## 六、宪章结语

> **这是一个交易平台，而不是策略脚本集合。**
> **这是一个长期系统，而不是一次性工具。**

本宪章用于帮助未来的设计与工程决策：

* 在复杂性出现之前，约束复杂性
* 在需求变化之前，守住边界

**当实现与宪章冲突时，应优先修正实现，而不是修改宪章。**

---

## Execution Context（执行上下文）设计

### 一、定位与职责（受宪章约束）

Execution Context 是**交易闭环中唯一有权对接真实/模拟市场成交的上下文**，其核心职责是：

* 接收 Strategy Context 产生的 **意图型指令（Intent）**
* 将意图转化为 **可执行的订单（Order）**
* 对接 Broker / Exchange Adapter
* 接收成交回报（Fill / Partial Fill / Reject / Cancel）
* 将**事实型结果**回推至 Position / Account / Strategy

Execution Context **不负责**：

* 资金是否合法（Account 决策）
* 仓位是否合理（Position 决策）
* 信号是否正确（Strategy 决策）

它只保证：**正确、可追溯、可重放地执行交易意图**。

---

### 二、核心对象模型

#### 1. ExecutionIntent（执行意图）

来自 Strategy Context 的上游输入：

* StrategyID
* AccountID
* InstrumentID
* Side（Buy / Sell）
* Qty / Price / OrderType
* TimeInForce
* RiskTags（只读，用于审计）

> Intent 是“我要做什么”，不是“我已经做了什么”。

#### 2. Order（执行订单）

Execution Context 内部生成：

* OrderID（全局唯一）
* BrokerOrderID（外部）
* State（New / Sent / PartiallyFilled / Filled / Canceled / Rejected）
* SubmittedAt / LastUpdatedAt

#### 3. ExecutionEvent（执行事件）

对外广播的**事实事件**：

* OrderAccepted
* OrderRejected
* OrderPartiallyFilled
* OrderFilled
* OrderCanceled

事件是**不可变的 append-only 流**。

---

### 三、文字版架构时序图（Execution 主链路）

```
[Strategy Context]
    |
    | 1. Generate ExecutionIntent
    v
[Execution Context]
    |
    | 2. Validate (syntax / state only)
    | 3. Build Order
    v
[Order Manager]
    |
    | 4. Send Order
    v
[Broker / Exchange Adapter]
    |
    | 5. Ack / Reject / Fill / Partial Fill
    v
[Execution Context]
    |
    | 6. Persist ExecutionEvent (event log)
    | 7. Update Order State Machine
    |
    +----------------------------+
    |                            |
    v                            v
[Position Context]        [Account Context]
(Update holdings)         (Update cash / margin)
    |
    v
[Strategy Context]
(Feedback / Metrics)
```

---

### 四、关键设计原则（不可违反）

1. **Execution 是事实层，不是决策层**

    * 不做风控裁决，只执行结果

2. **所有状态变化必须由 ExecutionEvent 驱动**

    * 禁止“直接改仓位 / 余额”

3. **Order 状态机必须单向、可重放**

    * 支持回测 / 仿真 / 实盘一致性

4. **外部不可信，内部强约束**

    * Broker 回报必须校验与归因

---

### 五、与其他 Context 的边界

* 与 Account Context：

    * Execution 只发送成交结果，不触碰余额计算逻辑

* 与 Instrument Context：

    * 只使用合约静态信息，不参与合约规则决策

* 与 Strategy Context：

    * Strategy 永远不知道订单“怎么被执行”，只知道“发生了什么”

---

### 六、为未来扩展预留的钩子

* 多 Broker 并行执行（Smart Routing）
* 拆单 / 算法单（TWAP / VWAP）
* 模拟撮合引擎（Backtest / Paper Trading）
* Execution Metrics（滑点、成交率、延迟）

---

> Execution Context 是整个平台最“冷静”的部分：
> **它不思考，只忠实记录世界发生了什么。**
