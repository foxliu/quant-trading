# StrategyContext 能力接口冻结说明（V1 · 宪章约束版）

> **定位再次声明**
> `StrategyContext` 不是资源集合，不是 Service Locator，
> 而是**策略在某一运行时刻被允许使用的能力白名单**。

---

## 一、能力设计的总原则（必须先记住）

### 三个「只允许」

1. **只允许能力，不允许资源**
2. **只允许视图，不允许实体**
3. **只允许表达意图，不允许直接执行**

如果某个方法违反其中任意一条，**直接否决**。

---

## 二、StrategyContext 冻结后的能力分组

我们只冻结「能力类别 + 方法语义」，
**不绑定返回结构、不绑定实现细节**。

---

## Ⅰ. 时间与运行态能力（Runtime Awareness）

### 1️⃣ 时间能力

```go
Now() Time
```

**语义裁定：**

* 返回“策略认为的当前时间”
* 回测 / 实盘由 Context 决定

**禁止：**

* 直接访问系统时间
* 依赖真实交易所时间

---

### 2️⃣ 运行模式感知（只读）

```go
Mode() RuntimeMode
```

* Backtest / Paper / Live
* **只允许读取，不允许分支业务逻辑**

> 用于日志、指标，而不是交易决策。

---

## Ⅱ. 行情与市场数据能力（Market View）

### 3️⃣ 行情读取（只读）

```go
GetBar(symbol, timeframe) BarView
```

```go
GetMarketSnapshot(symbol) MarketView
```

**裁定要点：**

* 返回的是 **View**
* 不承诺完整性、不承诺实时性

---

### 4️⃣ 历史数据访问（受控）

```go
GetHistory(symbol, timeframe, range) []BarView
```

**注意：**

* 这是能力，不是数据库
* 是否支持、支持多少，由 Context 决定

---

## Ⅲ. 持仓与账户“视图能力”（非资源）

⚠️ 这是最容易被污染的一组，边界必须非常清晰。

### 5️⃣ 持仓视图（只读）

```go
GetPositionView(symbol) PositionView
```

* View ≠ Position 实体
* 不允许修改
* 不保证与真实账户强一致

---

### 6️⃣ 资金摘要视图（只读）

```go
GetAccountSummary() AccountSummaryView
```

**明确禁止：**

* 获取账户 ID
* 获取账户实例
* 修改资金状态

---

## Ⅳ. 交易意图表达能力（最关键）

### 7️⃣ 提交交易意图（Intent）

```go
SubmitOrderIntent(intent OrderIntent) IntentResult
```

**这是 Context 中唯一“向外产生影响”的能力。**

语义裁定：

* Strategy **只表达“我想做什么”**
* 是否执行、何时执行、是否拆单：

    * 不由 Strategy 决定
    * 不由 Context 决定

---

### 8️⃣ 撤销 / 调整意图（可选）

```go
CancelIntent(intentID)
```

```go
ModifyIntent(intentID, newIntent)
```

> 是否支持，由具体 Context 决定。

---

## Ⅴ. 状态与记忆能力（策略私有）

### 9️⃣ 策略私有状态存取

```go
GetState(key) any
SetState(key, value)
```

**裁定：**

* 状态作用域 = Runtime
* 禁止跨策略共享
* 不保证持久化（由 Context 决定）

---

## Ⅵ. 日志与指标能力（观测性）

### 🔟 日志

```go
Log(level, message, fields...)
```

---

### 1️⃣1️⃣ 指标 / 统计

```go
EmitMetric(name, value, tags...)
```

**说明：**

* 只用于观测
* 不参与交易决策

---

## 三、明确被拒绝进入 Context 的能力（黑名单）

以下能力**永久禁止**出现在 `StrategyContext` 中：

* ❌ 下单执行结果（Fill / Trade）
* ❌ 账户实例 / Broker Client
* ❌ 资产类型判断（IsFuture / IsOption）
* ❌ 风控规则配置
* ❌ 账户选择逻辑
* ❌ 同步等待成交结果

---

## 四、冻结后的 StrategyContext “最终定义”

你可以把它理解为：

> **StrategyContext =
> 「时间 + 市场视图 + 账户视图 + 意图表达 + 私有记忆 + 观测能力」**

不多一项，也不少一项。

---

## 五、与主上下文时序图的强一致性校验

* Context 不主动执行任何动作 ✅
* Context 不持有核心资源 ✅
* Context 可被 Backtest / Live 替换 ✅
* Context 不破坏 Engine → Runtime → Strategy 闭环 ✅

---

## 六、重要冻结声明（请认真看）

> **从现在开始：**
>
> * 新策略只能使用以上能力
> * 新需求若需要新增能力，必须：
    >
    >   1. 证明其不破坏宪章
>   2. 证明其不污染主上下文
>   3. 经“设计层”显式扩展

否则，一律否决。
